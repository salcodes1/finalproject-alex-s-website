Post mortem

Yeah, when they say feature creep is real, they really mean it.

I have to be fair: I did have much higher ambitions for this iteration of the engine than I was able to achieve within the given timeframe. That doesn't mean I'm not proud of how the engine has shaped up so far. I think the Node-based architecture is solid and I'm quite proud of the game developer API I wrote around nodes interaction.

Where I would like to put in more effort the next couple of weeks would probably be on optimization, specifically trying to get rid of most "class" objects and replace them with structs. While the GC-managed lifetime of class objects has made iteration very quick, it has also burdened the game engine engineer in me, where I can never quite be sure if a certain Node or some other class object will ever get deallocated, either because there's some random reference to it stored somewhere, or because of some cyclical reference.

Additionally, I would probably put a lot more work into visual debugging tools, as I really could have used those. Perhaps tracy integration? It was quite trying to guess if the collision code really does not work, or if I've messed up the transform somehow and placed it far from where it's supposed to be.

Which brings me to my final potential place for improvement: defining scene hierarchies in lua would be amazing. I almost had everything implemented in Lua for this final project, but I had to cut it very late, since it was extremely buggy and I'm aiming for stability due to grading constraints.

Overall, I'm very happy with how the project turned out, and I look forward to improving it further in the future!
